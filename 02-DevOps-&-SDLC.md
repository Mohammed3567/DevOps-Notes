## What is SDLC?
A standard process followed by the software industry to design, develop,
and test software — with the end goal of delivering a high quality product.

---

## The 6 Phases

| Phase | What Happens |
|-------|-------------|
| **Planning** | Gather requirements and additional features needed |
| **Defining** | Document the final requirements formally |
| **Designing** | HLD (scalability, availability) + LLD (language, database choices) |
| **Building** | Developers write code, store it in version control (Git) |
| **Testing** | QE team deploys app on a test server, verifies functionality |
| **Deployment** | App promoted to production server for customers |

---

## Where DevOps Fits

DevOps Engineers are primarily involved in the **last 3 phases**:
Building → Testing → Deployment

Their job: **automate** these phases to improve speed and efficiency —
especially deployment, which was previously slow and manual.

---

## Key Insight
SDLC existed long before DevOps. DevOps didn't replace it —
it plugged into the last 3 phases and made them faster through automation.
